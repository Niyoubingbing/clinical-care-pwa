## 1. 产品概述

### 1.1 产品定位

**临床病人管理助手** 是一款面向临床医生的移动端 PWA 应用，用于管理住院病人的查房信息、待办事项和医嘱记录。

**核心价值**：
- 📱 **移动优先** - 手机上使用，随时随地记录
- ⚡ **秒开** - 本地存储，无网络也能用
- 🤖 **智能** - 自动解析待办时间，智能提醒
- 📋 **高效** - 批量导入病人，快捷操作

### 1.2 产品边界

**包含**：
- ✅ 病人信息管理（增删改查）
- ✅ 待办事项管理（智能时间解析 + 提醒）
- ✅ 查房列表（自定义排序）
- ✅ 批量导入/导出
- ✅ 快捷操作
- ✅ 每日小结

**不包含**（进阶功能，未来考虑）：
- ❌ 多用户协作
- ❌ 统计分析/报表
- ❌ 云端同步
- ❌ 原生 App

## 2. 用户画像

### 2.1 目标用户

| 属性 | 描述 |
|------|------|
| **角色** | 临床医生（住院医师/主治医师） |
| **科室** | 骨科（可扩展到其他外科科室） |
| **技术水平** | 熟练使用智能手机 |
| **使用场景** | 查房时、办公室、家中 |

### 2.2 用户痛点

1. **病人多，容易漏** - 40+ 病人，每天哪些要查血、哪些要换药，容易忘记
2. **待办分散** - 每个病人的待办不同，难以统一管理
3. **时间感知弱** - "星期二出院"这种待办，到了当天容易忘记
4. **数据录入慢** - 每天要重新输入病人信息，效率低

### 2.3 用户目标

1. **快速记录** - 查房时快速记录每个病人的待办
2. **智能提醒** - 系统自动提醒今天要做的事
3. **高效导入** - 粘贴最短文本，快速同步全量数据
4. **每日小结** - 快速生成当天完成的事项，用于交班/记录

## 3. 核心工作流

### 3.1 每日工作流

```
早晨 8:00
  ↓
打开 App，查看今日提醒
（查血、换药、待办到期）
  ↓
查房（按自定义顺序）
  ↓
给每个病人添加/更新待办
（系统自动解析时间，到期提醒）
  ↓
查房结束，生成每日小结
（复制给上级/交班）
  ↓
随时查看待办，完成打勾
```

### 3.2 每周工作流

```
周一
  ↓
批量导入本周病人
（粘贴文本，自动同步）
  ↓
每天查房，更新待办
  ↓
周五
  ↓
导出数据备份
（防止丢失）
```

## 4. 功能需求

### 4.1 病人管理

#### 4.1.1 病人列表

**功能描述**：
- 显示所有病人，按**自定义查房顺序**排序
- 支持**分组筛选**（如：解组、勇组）
- 支持**搜索**（床号、姓名、诊断）
- 每个病人显示：
  - 床号、姓名、诊断
  - 待办数量（未完成）
  - 查血/换药状态（逾期高亮）
  - 分组颜色标签

**交互细节**：
- 点击病人 → 进入详情页
- 长按病人 → 快速操作菜单（删除、分组）
- 搜索支持**拼音首字母**（如：输入 "wyx" 搜索 "魏亿鑫"）

**验收标准**：
- [ ] 列表加载时间 < 500ms（40 个病人）
- [ ] 搜索响应时间 < 100ms
- [ ] 拼音搜索准确率 > 95%

---

#### 4.1.2 添加/编辑病人

**功能描述**：
- 支持手动添加病人
- 支持编辑病人信息：
  - 床号
  - 姓名
  - 诊断
  - 分组
  - 手术日期
  - 换药频率（天）
  - 上次换药日期
  - 查血日期（周几）

**交互细节**：
- 编辑界面使用 **Bottom Sheet**（移动端友好）
- 分组支持**自定义**（添加新分组、修改分组名称/颜色）
- 日期选择器使用原生日期组件

**验收标准**：
- [ ] 保存后立即更新列表
- [ ] 分组修改后，列表颜色实时更新
- [ ] 支持删除病人（带撤销）

---

#### 4.1.3 批量导入病人

**功能描述**：
- 支持粘贴文本批量导入
- 文本格式：`床号 姓名 诊断`（如：`309W23 魏亿鑫 取除骨折内固定装置`）
- 导入逻辑：
  - 如果病人已存在（同名）→ **更新**诊断信息
  - 如果病人不存在 → **新增**
  - 如果导入文本中没有某个病人 → **删除**（弹窗确认）

**交互细节**：
- 导入前显示**预览**（哪些新增、哪些更新、哪些删除）
- 删除操作需要**二次确认**
- 导入后显示**结果统计**（新增 X 人，更新 X 人，删除 X 人）

**验收标准**：
- [ ] 导入 40 个病人 < 3 秒
- [ ] 同名病人正确更新（不重复）
- [ ] 删除确认弹窗清晰明了

---

### 4.2 待办管理

#### 4.2.1 添加待办

**功能描述**：
- 每个病人可以有多个待办
- 待办内容：**自由文本**（如："星期二出院"、"后天换药"、"周四复查X光"）
- 待办类型（可选）：换药、开术前、明天出院、康复会诊、会诊、复查、开查血、其他

**智能时间解析**（核心功能）：
- 自动识别待办文本中的时间信息：
  - **星期几**（周一~周日、星期一到星期日）
  - **今明后天**（今天、明天、后天）
  - **具体日期**（如：12月5日、12/5、2026-12-05）
- 解析成功后：
  - 计算**剩余天数**
  - 存储**目标日期**
  - 到了当天，**高亮提醒**

**交互细节**：
- 添加待办时，**实时解析**时间（输入时显示解析结果）
- 如果解析失败，提示用户"未识别到时间"
- 支持**快捷待办**（常用待办一键添加，如：换药、查血）

**验收标准**：
- [ ] 时间解析准确率 > 90%
- [ ] 解析结果实时显示（< 300ms）
- [ ] 支持"今明后天"、"星期几"、"具体日期"三种格式

---

#### 4.2.2 待办列表

**功能描述**：
- 显示所有病人的待办（按时间排序）
- 支持**筛选**：
  - 全部
  - 未完成
  - 已完成
  - 今天到期
  - 已逾期

**交互细节**：
- 每个待办显示：
  - 病人床号、姓名
  - 待办内容
  - 剩余天数（如：还剩 2 天、今天到期、已逾期 3 天）
  - 状态（未完成/已完成）
- 点击待办 → 跳转到病人详情
- 左滑待办 → 快速操作（完成、删除）

**验收标准**：
- [ ] 待办列表加载 < 500ms
- [ ] 到期待办**高亮显示**（红色/橙色）
- [ ] 完成待办后，列表实时更新

---

#### 4.2.3 智能提醒

**功能描述**：
- **启动时检查**（每次打开 App）：
  - 检查所有待办，找出**今天到期**和**已逾期**的
  - 在首页**高亮显示**
  - 显示**提醒数量**（如：3 个待办今天到期）

- **定时检查**（可选）：
  - 每隔 1 小时检查一次
  - 如果有新的到期待办，发送**浏览器通知**（如果授权）

**交互细节**：
- 首页顶部显示**提醒栏**：
  - "今天有 3 个待办到期"
  - "有 2 个待办已逾期"
- 点击提醒栏 → 跳转到待办列表（筛选：今天到期/已逾期）

**验收标准**：
- [ ] 启动时检查 < 1 秒（40 个病人）
- [ ] 提醒栏显示正确
- [ ] 浏览器通知正常发送（如果授权）

## 5. 交互设计

### 5.1 导航结构

```
底部 Tab 导航
  ├── 查房（首页）
  ├── 待办
  ├── 设置
```

```
病人详情页
  ↓
```

```
待办列表
  ↓
```

## 6. 技术要求

### 6.1 性能要求

| 指标 | 要求 |
|------|------|
| 首次加载 | < 2 秒（PWA 缓存后 < 500ms） |
| 列表滚动 | 60fps（无卡顿） |
| 操作响应 | < 200ms |
| 数据导入（40 人） | < 3 秒 |

### 6.2 离线要求

- ✅ 首次加载后，所有功能**无网络可用**
- ✅ 数据存储在 **IndexedDB**（本地）
- ✅ PWA Service Worker 缓存静态资源

### 6.3 兼容性要求

| 平台 | 浏览器 | 支持 |
|------|--------|------|
| iOS | Safari | ✅ |
| iOS | Chrome | ✅ |
| Android | Chrome | ✅ |
| Android | Edge | ✅ |
| 桌面 | Chrome/Edge | ✅ |

### 6.4 数据要求

- **存储方案**：IndexedDB（通过 Dexie.js）
- **数据备份**：导出 JSON（手动）
- **数据恢复**：导入 JSON（手动）
- **数据安全**：本地存储，无云端同步
